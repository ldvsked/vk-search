# VK Search

Командный проект в рамках VK Education.

Система полнотекстового поиска по публикациям VK с использованием OpenSearch и RAG-подхода для генерации кратких ответов через LLM.

## Описание проекта

Сервис предназначен для поиска информации по публикациям VK и получения кратких ответов на основе найденных данных.

Проект состоит из двух функциональных блоков:

* **Data Pipeline (Python):** сбор, очистка, разбиение постов на чанки и индексация данных в OpenSearch.
* **Backend API (Go):** серверная часть для обработки пользовательских запросов, аутентификации, взаимодействия с поисковым индексом и LLM-сервисом.

## Мой вклад

Backend-разработка на Go:

* разработка REST API для поиска и управления доступом;
* реализация аутентификации и авторизации пользователей с использованием JWT;
* интеграция с OpenSearch для полнотекстового поиска;
* интеграция с LLM API для генерации суммаризаций в рамках RAG-подхода;
* интеграция с PostgreSQL и настройка запуска сервисов через Docker Compose.

## MVP

* **Парсинг:** сбор данных с информационных ресурсов.
* **Индексация:** загрузка очищенных данных в OpenSearch с использованием BM25.
* **Backend:** REST API для поиска и управления доступом.
* **RAG/LLM:** генерация кратких ответов на основе найденных документов.

## Запуск проекта

Для развертывания приложения VK Search выполните приведенные ниже команды в терминале.

### 1. Клонирование репозитория

Скопируйте проект локально и перейдите в созданную директорию:

```bash
git clone [https://github.com/andreym1234/vk-search.git](https://github.com/andreym1234/vk-search.git)
cd vk-search

```

### 2. Настройка окружения

Проект требует настройки переменных окружения. Используйте предоставленный шаблон:

1. Создайте файл `.env` из примера:

```bash
cp .env.example .env

```

2. Откройте файл `.env` в текстовом редакторе и укажите необходимые параметры:

* `JWT_SECRET`: Установите надежный ключ для шифрования токенов.
* `LLM_API_KEY`: Укажите ваш API-ключ для OpenRouter (необходимо для работы LLM).
* `VK_TOKEN`: Токен доступа к API VK (требуется только для функции сбора новых данных).

### 3. Запуск приложения

Развертывание и сборка контейнеров осуществляются через Docker Compose:

```bash
docker-compose up -d --build

```

**Примечание:**
При первом запуске контейнер `data_bootstrap` автоматически наполнит базы данных начальными (seed) данными из директории `data/seed/`. Этот процесс может занять некоторое время.

### 4. Проверка статуса

Чтобы убедиться, что все сервисы запущены корректно, выполните:

```bash
docker-compose ps

```

Все контейнеры (`vk_search_backend`, `vk_search_postgres`, `vk_search_opensearch`) должны иметь статус **Up**.

### Остановка проекта

Для завершения работы сервисов выполните:

```bash
docker-compose down

```

Если вы хотите полностью очистить данные (удалить тома с БД), используйте:

```bash
docker-compose down -v

```

## API и безопасность

### Аутентификация

* `POST /auth/login` — получение JWT-токена.
* Система использует ролевую модель доступа (RBAC) для разграничения прав пользователей.

### Поиск и LLM

* `GET /search` — полнотекстовый поиск по базе постов (BM25).
* `POST /ask` — суммаризация результатов поиска с помощью LLM (RAG-подход).

### Документы и администрирование

* `GET /documents/{id}` — получение полного текста поста.
* `GET /admin/stats` — статистика системы (индексация, логи).
* `GET /health` — проверка статуса сервисов.

## Логирование

Все запросы к методам `/search` и `/ask` автоматически фиксируются в таблице `search_logs` базы данных PostgreSQL для аудита действий пользователей.
